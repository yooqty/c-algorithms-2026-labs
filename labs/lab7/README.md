Преобразование кода ДНК в сжатый вид.

Входной файл в этом каталоге содержит ASCII-запись последовательности азотистых оснований в молекуле ДНК 13-й хромосомы гена человека. Предполагается, что эта последовательность отвечает за наследственные признаки человека.
В каждой строке по 70 остатков, обозначенных буквами A, G, C, T или N (аденин, гуанин, цитозин, тимин) и последний символ N — неопределённый тип.

Для сжатия записи каждый символ нужно заменить 3-х битной последовательностью. На входе символы A, G, C, T, N с кодами 0101, 0107, 0103, 0124 и 0116 (восьмеричные числа).
Также в конце строки встречается символ перехода на новую строку с кодом 012.
Последняя строка может быть неполной (иметь менее 70 символов).

Прочитать файл, упаковать заново без символов 012 в данной последовательности и записать в файл в двоичном виде как множество чисел типа long long int.

Установить соответствие значение триады битов азотистому основанию так:

(0 0 0) = N  
(0 0 1) = A  
(0 1 0) = C  
(1 0 0) = G  
(1 1 1) = T

Программа также должна подсчитать количество каждого из азотистых оснований и вывести эти значения в десятичном виде.

Предусмотреть возможность использования других файлов для сжатия (у человека 23 хромосомы) с передачей имени файла через параметр командной строки argv[1].

Для 4-х типов азотистых оснований хватило бы 2-х битов, но вот N мешает... Попытайтесь найти алгоритм, в котором сжатие использует 2-х битовую кодировку.
